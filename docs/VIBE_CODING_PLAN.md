# Vibe Coding 学习与实践计划 v0.1

## 1. 学习目标
通过开发这款个人知识库笔记软件，学习如何把 AI 当作结对工程伙伴，完成从想法、需求、设计、原型、实现、测试到迭代的完整流程。

## 2. 工作方式

### 2.1 每轮开发固定节奏
1. 描述目标：这轮要完成什么。
2. 让 AI 读上下文：代码、设计、需求、错误信息。
3. 要求 AI 给出方案或直接实现。
4. 人类审查：确认是否符合产品直觉。
5. 运行验证：启动、测试、截图、手动试用。
6. 记录经验：把问题和决策写回项目文档。

### 2.2 提示词模板

#### 需求澄清
```text
我们正在做一个跨端个人知识库笔记软件。
请基于当前 PRD，帮我拆解这个功能的用户场景、边界条件、MVP 范围和验收标准。
功能：{功能名称}
```

#### 代码实现
```text
请先阅读当前项目结构和相关文件，然后实现 {功能名称}。
要求：
1. 遵循现有代码风格。
2. 不做无关重构。
3. 完成后运行可用的测试或启动检查。
4. 最后说明改了哪些文件、如何验证。
```

#### Bug 修复
```text
当前出现问题：{问题描述}
复现步骤：{步骤}
期望结果：{期望}
实际结果：{实际}
请定位原因、修复，并补充必要测试。
```

#### 设计迭代
```text
请基于当前 UCD，优化 {页面/组件} 的交互和视觉。
要求：
1. 保持 macOS 风格。
2. 保持简洁，避免功能堆叠。
3. 输出具体布局、状态、交互细节。
```

## 3. 阶段计划

### 阶段 1：需求与设计
- 输出 PRD。
- 输出 UCD。
- 明确 MVP。
- 决定技术栈。

### 阶段 2：可运行原型
- 创建桌面端项目。
- 技术栈采用 Tauri 2 + React + TypeScript。
- 搭建三栏布局。
- 实现假数据笔记列表。
- 实现编辑器基础输入。

### 阶段 3：真实数据
- 接入本地文件夹。
- 接入绿联云 NAS 挂载目录。
- 支持 Markdown 文件读写。
- 建立索引数据库。
- 实现搜索。

### 阶段 4：知识库能力
- 实现 `[[引用]]`。
- 实现反向链接。
- 实现标签。
- 实现 MOC 页面模板。

### 阶段 5：跨端与打包
- 完成 Windows 打包。
- 设计 iPhone 移动端。
- 决定移动端方案。
- 增加备份和冲突处理。

## 4. 关键学习点

- 如何把模糊想法变成 PRD。
- 如何让 AI 帮你拆 MVP。
- 如何用 AI 快速搭建前端界面。
- 如何让 AI 阅读并修改已有代码。
- 如何用测试和截图检查 AI 生成结果。
- 如何控制需求范围，避免越做越复杂。
- 如何设计本地优先软件的数据模型。

## 5. 每日练习建议

每天只做一个小闭环：
- 一个功能点。
- 一次可运行验证。
- 一条项目记录。

示例：
- 今天只做“新建笔记”。
- 今天只做“搜索弹窗”。
- 今天只做“NAS 路径选择”。
- 今天只做“代码块样式”。
